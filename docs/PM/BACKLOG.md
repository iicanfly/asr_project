# BACKLOG

## 维护原则
- 本项目所有开发任务默认都属于`双环境`任务。
- 外网用于开发与调试，内网是最终部署环境。
- 任何修改都必须同时考虑内网与外网行为是否一致可控。
- 由于当前无法直接在内网调试，所有实现需要优先保证内网路径的稳妥性，避免破坏已验证通过的内网逻辑。

## 当前优先级

### P0：实时转写质量提升
#### 1. 静音音频优化
- 目标：旁边人小声说话、低价值背景音、不应进入转写主流程。
- 原因：当前实时转写会误收环境中的微弱无效声音，影响质量和稳定性。
- 网络范围：双环境。
- 验收标准：明显的低音量背景声不再频繁触发转写输出。

#### 2. 语气词过滤
- 目标：过滤“嗯”“啊”“呀”等低信息量语气词。
- 原因：当前输出中语气词过多，影响文本质量。
- 网络范围：双环境。
- 验收标准：常见语气词显著减少，且不误伤正常短句。

#### 3. 多层次音频转写
- 目标：前端先显示小段实时结果，随后在更大音频片段形成后，用更高质量结果替换前面的粗糙结果。
- 原因：纯小段实时转写虽然快，但错误率高，需要通过大段复写提升整体可读性与准确率。
- 网络范围：双环境。
- 验收标准：前端能看到“先出结果、后被更高质量结果替换”的稳定行为。
- 当前状态：
  - 基础协议与最小闭环已开始落地，下一步重点变成“真实录音效果调优 + 分段边界配合验证”。

#### 4. 转写文本分段策略
- 目标：避免点击开始录音后所有转写内容一直堆在同一个段落中，同时避免分段过碎。
- 原因：当前没有分段机制，导致内容结构差；但过于频繁分段也会降低可读性。
- 网络范围：双环境。
- 验收标准：转写结果具备合理段落边界，能兼顾连续性与可读性。
- 当前状态：
  - 已开始从“纯时间窗合并”转向“后端 `segment_id` + 句子边界启发式 + 前端智能拼接”的组合策略。
  - 下一步重点变成真实录音下的边界调优，以及观察是否仍有过碎或过黏现象。

## 暂缓事项
- 音频上传功能当前可用，但暂不作为近期重点优化对象。
- 文档生成功能当前可用，但暂不作为近期重点优化对象。

## 当前阶段目标
- 在不破坏现有内外网调用链路的前提下，优先大幅提高实时转写质量。

## 2026-05-10 进度补充
- 长时间录音稳定性已经开始进入第一轮代码级防护：
  - 优先削减前端长时间录音时不必要的内存占用与缓存写放大。
- 停止录音收尾链路已经开始补强：
  - 优先减少“停止太快导致最后一小段尾音丢失”的问题。
- 本地缓存容量保护已经开始补强：
  - 优先降低缓存超限或缓存损坏后导致前端恢复异常的风险。
- 双层转写结果去重已经开始补强：
  - 优先减少无意义的重复 partial / rewrite 回推。
- 前端结果投递去重已经开始补强：
  - 优先减少同一 `result_id` 重复到达时的重复渲染。
- 前端消息 ID 一致性已经开始补强：
  - 优先减少缓存恢复后新旧消息 ID 碰撞的风险。
- 静音过滤 / 弱语音优化已进入第二轮规则细化：
  - 不再让 `active_ratio` 单独放行实时切片，而是增加“最小有声时长 / 最小活跃时长 / 柔和语音兜底”的组合门槛。
  - 下一步重点变成真实录音校准：确认误抑制旁边小声说话的同时，没有把主说话人的轻声内容一起杀掉。
  - 尾静音路径也已开始单独收紧：短促弱语音在尾静音到来时将更容易被直接丢弃，而不是被切成一条独立转写。
  - 已补上离线 wav 回放工具，但当前本地 3 份 wav 样本都偏强主语音；下一步仍需要更贴近“弱背景插话”的真实样本来继续校准。
  - 已补上环境变量调参路径；下一步可优先通过 `ONLINE_REALTIME_*` 做外网阈值试验，再决定是否同步内网。
  - 已补上 gain sweep 分析能力；下一步可把真实主语音样本与真实背景插话样本做同档位衰减对照，寻找更稳的分界。
  - 已补上事件级时间线回放能力：可以直接看到每次 `process / drop / stop flush` 发生在第几秒、对应哪段 buffer、命中了哪条 `speech_gate / tail_gate`，方便明早拿真实录音快速定位误触发点。
  - 已补上前景/背景混音回放能力：可把一条主语音 wav 与一条弱背景 wav 按不同增益、偏移时间混合回放，用于近似模拟“旁边人小声插话”并观察门槛分流表现。
  - 已补上主音频 / 背景音频片段裁剪能力：可只截取某几秒的短片段再分析，更方便逼近“短促弱插话”“尾段短背景说话”等具体场景。
  - 已补上多偏移场景扫描与 JSON 导出：可批量比较不同背景插话时机，并把结构化结果落盘，便于明早做多组场景对照。
  - 已开始把离线结论回灌到线上规则：新增“连续有声时长 / 连续活跃时长”约束，专门压制总量看起来够、但实际上只是碎片化弱插话的片段。
  - 已补第二道文本闸门：针对无人说话时 ASR 偶发产出的“嗯 / yeah / thank you / 你好 / what's that”这类低信息幻觉短句，发送前会做整句级片段清洗与拦截。
  - 已补“保守上传门控”：对边界处疑似短语音，不再立刻按静音片段上传或清空，而是先保留一小段缓冲等待后续上下文确认。
  - 该门控默认按“兼容后续大段回写”设计：优先减少无效静音上传，但不直接把疑似短语音从原始缓冲链路中粗暴删除。
  - 下一步重点变成真实麦克风回归：确认静音时幻觉串明显下降，同时不要误删真实的短英文表达或礼貌短句。
  - 已正式进入“简化重构实施”阶段：
    - 先新增 simplified realtime pipeline 开关，不直接删除 legacy 路径。
    - 外网开发模式下，新的默认实时参数已切到更快 partial 基线：
      - `min_audio_seconds=0.6`
      - `chunk_seconds=2.5`
      - `max_audio_seconds=12.0`
      - `stop_flush_min_seconds=0.25`
    - 下一步重点不再是继续补尾巴 patch，而是观察：
      - partial 是否明显更快
      - 有话是否更稳定能出
      - rewrite 是否仍兼容
  - 语气词清洗策略已进一步收敛为“最小可控版”：
    - 只删除独立片段中的语气词 / 明显幻觉词；
    - 其余文字保持不变；
    - 不再因为一句话中夹了语气词，就把整句整体过滤掉。
