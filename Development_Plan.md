# 项目开发计划 (Development Plan) - 调整版 (2026-04-22)

本计划已根据 **[最小改动原则]** 和 **[OpenAI 兼容协议标准化]** 进行调整。目标是先在外网环境调通 Qwen ASR/LLM，随后通过修改配置文件实现内网无缝迁移。

| 角色 | 负责内容 |

| **AI (我)** | OpenAI 协议适配层代码编写、双模式配置重构、实时转写转发逻辑、会议纪要生成逻辑、离线打包脚本生成。 |

---

## 2. 详细开发阶段

### 第一阶段：开发环境与依赖准备 (由人类执行)
*   [x] **任务 1.1**：安装 Node.js (v18.20.8)。
*   [x] **任务 1.2**：安装 Miniconda 并配置 E 盘环境。
*   [x] **任务 1.3**：创建 `voice-system` Python 3.9 环境。
*   [ ] **任务 1.4**：在 Conda 环境中安装新依赖：`pip install openai aiohttp python-docx`。
*   [ ] **任务 1.5**：获取阿里云 DashScope API Key 并填入 `config.py`（外网调试用）。

### 第二阶段：OpenAI 协议适配与双模式重构 (由 AI 执行)
*   [ ] **任务 2.1**：重构 `config.py`，支持 `USE_INTRANET` 开关及 OpenAI 兼容参数。
*   [ ] **任务 2.2**：在 `main.py` 中引入 OpenAI SDK，实现 ASR 实时流式转发（对接 qwen3-asr-flash-2026-02-10）。
*   [ ] **任务 2.3**：在 `main.py` 中实现 LLM 会议纪要生成逻辑（对接 qwen-plus/qwen3.5）。
*   [ ] **任务 2.4**：完善 `index.html` 中的 WebSocket 回显逻辑，确保文字实时上屏。

### 第三阶段：功能增强与容灾补偿 (由 AI 执行)
*   [ ] **任务 3.1**：完善“录音缓存与补偿机制”，支持断网/服务异常时的音频暂存与事后补发。
*   [ ] **任务 3.2**：实现会议纪要预览及导出为 Word (.docx) 功能。
*   [ ] **任务 3.3**：优化音频上传调试模块，支持大文件进度条显示。

### 第四阶段：离线打包与内网迁移 (由 AI & 人类协作)
*   [ ] **任务 4.1 (AI)**：编写 `conda-pack` 后端环境打包脚本。
*   [ ] **任务 4.2 (AI)**：编写 `electron-builder` 前端一键打包配置。
*   [ ] **任务 4.3 (人类)**：将打包后的环境迁移至内网 Win7 机器，修改 `config.py` 中的 `USE_INTRANET = True` 并填入私有化部署地址进行测试。

## 3. 里程碑 (Milestones)

1.  **M1 (外网调通)**：在外网环境下，实现语音实时转文字、自动生成会议纪要。
2.  **M2 (协议标准)**：代码逻辑完全适配 OpenAI 规范，具备一键切换内网的能力。
3.  **M3 (功能完备)**：具备容灾补偿、Word 导出、文件上传调试等全套功能。
4.  **M4 (交付就绪)**：产出可独立运行的离线安装包及运维手册。
