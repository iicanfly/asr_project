## 方案一：Web 服务（多用户通过 IP 访问）

### 适用场景

- 多人共享使用
- 服务器部署，客户端通过浏览器访问
- 无需安装客户端软件

### 部署架构

```
┌─────────────────────────────────────────────────────────────┐
│                       服务器电脑                              │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              后端服务 (Python/FastAPI)              │    │
│  │              监听: 0.0.0.0:8000                      │    │
│  └───────────────────┬─────────────────────────────────┘    │
│                      │                                       │
│  ┌───────────────────▼─────────────────────────────────┐    │
│  │               内网 AI 服务                           │    │
│  │               (ASR + LLM)                            │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                           ▲
                           │ HTTP/WebSocket
                           │
┌──────────────────────────┴──────────────────────────────────┐
│                        客户端电脑                             │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              浏览器 (Chrome/Edge)                    │    │
│  │         http://服务器IP:3000                         │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 部署步骤

#### 步骤 1：配置 AI 服务地址

编辑服务器电脑上的 `config.py` 文件：

```python
# 切换为内网模式
USE_INTRANET = True

# 配置内网 AI 服务器地址
BASE_URL = "http://你们的AI服务器IP:端口/v1"

# 配置 API Key（如果需要）
API_KEY = "你们的API_KEY"

# 配置模型名称
ASR_MODEL = "你们的ASR模型名"
LLM_MODEL = "你们的LLM模型名"
```

#### 步骤 2：修改后端监听地址

编辑 `main.py` 文件最后一行：

```python
# 原来（只允许本机访问）
uvicorn.run(app, host="127.0.0.1", port=8000)

# 改为（允许外网访问）
uvicorn.run(app, host="0.0.0.0", port=8000)
```

#### 步骤 3：启动后端服务

在服务器电脑上：

```bash
cd /path/to/project
python main.py
```

后端服务将启动在 `http://0.0.0.0:8000`，即监听所有网络接口。

#### 步骤 4：启动前端服务

在服务器电脑上（新开一个终端）：

```bash
cd /path/to/project
npm start
```

前端服务将启动在 `http://localhost:3000`

#### 步骤 5：配置防火墙

确保服务器电脑防火墙允许以下端口的入站连接：

| 端口 | 协议 | 说明 |
|------|------|------|
| 3000 | TCP | 前端服务 |
| 8000 | TCP | 后端 API |

**Windows 防火墙配置示例**：

```powershell
# 添加防火墙规则
netsh advfirewall firewall add rule name="Voice Backend" dir=in action=allow protocol=TCP localport=8000
netsh advfirewall firewall add rule name="Voice Frontend" dir=in action=allow protocol=TCP localport=3000
```

#### 步骤 6：客户端访问

在其他电脑的浏览器中访问：

```
http://服务器IP:3000
```

例如，如果服务器 IP 是 `192.168.1.100`：

```
http://192.168.1.100:3000
```

#### 步骤 7：客户端配置后端地址

1. 打开系统配置页面（点击左侧 "⚙️ 系统配置"）
2. 在"后端服务器地址"中，将 `http://127.0.0.1:8000` 改为 `http://192.168.1.100:8000`
3. 点击 "💾 保存配置"
4. 点击 "🔍 测试连接"，确认显示 ✅ 连接成功

### 多用户同时使用

- 支持：多个用户同时访问、录音、生成文档
- 每个用户的会话独立，互不干扰
- 历史文档保存在各自浏览器的 localStorage 中

### 开机自启动（Windows）

创建启动脚本 `start_server.bat`：

```batch
@echo off
cd /d "C:\path\to\project"
start /B python main.py
timeout /t 3 /nobreak > nul
start npm start
```

将快捷方式放到启动文件夹：
```
C:\Users\用户名\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\
```

### 网络要求

| 要求 | 说明 |
|------|------|
| 局域网 | 服务器和客户端在同一局域网 |
| 延迟 | 建议 < 50ms |
| 带宽 | 建议 > 100Mbps |

## 方案二：EXE 桌面应用

### 适用场景

- 单用户独立使用
- 无需浏览器访问
- 内网环境，无法访问外网

### 部署架构

```
┌─────────────────────────────────────┐
│         桌面应用 (EXE)               │
│  ┌─────────────────────────────┐    │
│  │    Electron 前端界面         │    │
│  └───────────┬─────────────────┘    │
│              │                       │
│  ┌───────────▼─────────────────┐    │
│  │  voice_backend.exe (后端)   │    │
│  └───────────┬─────────────────┘    │
│              │                       │
│  ┌───────────▼─────────────────┐    │
│  │   内网 AI 服务              │    │
│  │   (ASR + LLM)               │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

### 部署步骤

#### 步骤 1：配置 AI 服务地址

编辑 `config.py` 文件：

```python
# 切换为内网模式
USE_INTRANET = True

# 配置内网 AI 服务器地址
BASE_URL = "http://你们的AI服务器IP:端口/v1"

# 配置 API Key（如果需要）
API_KEY = "你们的API_KEY"

# 配置模型名称
ASR_MODEL = "你们的ASR模型名"
LLM_MODEL = "你们的LLM模型名"
```

#### 步骤 2：安装依赖

```bash
# 安装 Node.js 依赖
npm install

# 安装 Python 打包工具
pip install pyinstaller
```

#### 步骤 3：执行打包

```bash
npm run dist
```

打包过程：
1. 使用 PyInstaller 将 Python 后端打包为 `voice_backend.exe`
2. 使用 Electron Builder 将前端打包为安装程序

#### 步骤 4：获取安装包

打包完成后，在 `dist/` 目录下生成：

```
dist/
└── 内网语音纪要系统 Setup 0.1.0.exe
```

### 用户安装和使用

#### 安装

1. 双击 `内网语音纪要系统 Setup 0.1.0.exe`
2. 选择安装目录（默认：`C:\Users\用户名\AppData\Local\Programs\内网语音纪要系统\`）
3. 等待安装完成
4. 桌面出现快捷方式

#### 使用

1. 双击桌面快捷方式启动应用
2. 应用自动启动后端服务
3. 开始录音/上传音频，系统自动识别并生成文档

### 配置文件位置（高级用户）

如果需要修改 AI 服务地址，配置文件位于：

```
安装目录\resources\app.asar.unpacked\config.py
```

### 卸载

1. 打开 Windows 控制面板
2. 选择 "卸载程序"
3. 找到 "内网语音纪要系统" 并卸载

## 方案三：API 接口（供 OA 调用）

### 适用场景

- 集成到现有系统（OA、ERP 等）
- 批量处理音频文件
- 自定义前端界面

### 接口列表

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/v1/debug/status` | GET | 检查服务状态 |
| `/api/v1/audio/upload` | POST | 上传音频文件识别 |
| `/api/v1/asr/stream` | WebSocket | 实时语音流识别 |
| `/api/v1/llm/summarize` | POST | 生成文档 |
| `/api/v1/llm/regenerate` | POST | 优化重新生成 |
| `/api/v1/export/word` | POST | 导出 Word 文档 |
| `/api/v1/export/download/{filename}` | GET | 下载生成的文件 |

### 接口调用示例

#### 1. 检查服务状态

```python
import requests

response = requests.get('http://后端服务器:8000/api/v1/debug/status')
print(response.json())
# 输出: {"status": "ok", "asr_model": "qwen-asr", "llm_model": "qwen2.5-7b"}
```

#### 2. 上传音频文件识别

```python
import requests

# 上传音频文件
files = {'file': open('meeting.wav', 'rb')}
response = requests.post('http://后端服务器:8000/api/v1/audio/upload', files=files)
result = response.json()

print(result['transcript'])  # 识别结果
print(result['speaker_id'])  # 说话人
```

#### 3. 生成会议纪要

```python
import requests

# 准备转写数据
data = {
    "transcript": [
        {"speaker": "张三", "content": "今天我们讨论了项目进度", "time": "10:00:00"},
        {"speaker": "李四", "content": "前端开发已经完成80%", "time": "10:01:00"}
    ],
    "doc_type": "meeting"  # meeting / report / publicity
}

# 生成文档
response = requests.post(
    'http://后端服务器:8000/api/v1/llm/summarize',
    json=data
)
result = response.json()

print(result['doc_title'])      # 文档标题
print(result['summary_text'])   # 文档内容
```

#### 4. 优化重新生成

```python
import requests

data = {

    "doc_type": "meeting",
    "feedback": "请使用更正式的商务语言，详细展开每个议题"

response = requests.post(
    'http://后端服务器:8000/api/v1/llm/regenerate',
```

#### 5. 导出 Word 文档

```python

    "summary": "文档内容...",
    "transcript": [...],
```

### Java 调用示例

```java
import java.io.*;
import java.net.http.*;
import org.json.*;

public class VoiceApiClient {
    private static final String BASE_URL = "http://后端服务器:8000";

    // 生成会议纪要
    public static String generateMinutes(String transcriptJson) throws Exception {
        HttpClient client = HttpClient.newHttpClient();

        // 构建 JSON 请求体
        JSONObject requestBody = new JSONObject();
        requestBody.put("transcript", new JSONArray(transcriptJson));
        requestBody.put("doc_type", "meeting");

        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create(BASE_URL + "/api/v1/llm/summarize"))
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(requestBody.toString()))
            .build();

        HttpResponse<String> response = client.send(request,
            HttpResponse.BodyHandlers.ofString());

        JSONObject result = new JSONObject(response.body());
        return result.getString("summary_text");
    }
}
```

### C# 调用示例

```csharp
using System;
using System.Net.Http;
using System.Text;
using System.Threading.Tasks;
using Newtonsoft.Json;

public class VoiceApiClient
{
    private static readonly HttpClient client = new HttpClient();
    private const string BaseUrl = "http://后端服务器:8000";

    public async Task<string> GenerateMinutesAsync(string transcriptJson)
    {
        var requestBody = new
        {
            transcript = JsonConvert.DeserializeObject<object[]>(transcriptJson),
            doc_type = "meeting"
        };

        var json = JsonConvert.SerializeObject(requestBody);
        var content = new StringContent(json, Encoding.UTF8, "application/json");

        var response = await client.PostAsync($"{BaseUrl}/api/v1/llm/summarize", content);
        var responseString = await response.Content.ReadAsStringAsync();

        dynamic result = JsonConvert.DeserializeObject(responseString);
        return result.summary_text;
    }
}
```

## 常见问题

### Q1: 打包后 EXE 文件很大（200-300MB）？

**A**: 这是正常的，包含了：
- Electron 运行时（~100MB）
- Python 运行时（~50MB）
- 依赖库（~100MB）

### Q2: 打包后运行报错找不到 AI 服务？

**A**: 检查 `config.py` 中的 `BASE_URL` 是否正确指向内网 AI 服务器。

### Q3: API 调用超时怎么办？

**A**: 语音识别和文档生成需要时间，建议：
- 客户端设置较长超时时间（30-60秒）
- 使用异步方式处理

### Q4: 如何限制只有内网可以访问？

**A**: 
1. 后端只监听内网 IP：`uvicorn.run(app, host="内网IP", port=8000)`
2. 配置防火墙规则
3. 添加 API 认证机制

### Q5: 支持并发识别吗？

**A**: 支持，但需要确保 AI 服务有足够的并发处理能力。
